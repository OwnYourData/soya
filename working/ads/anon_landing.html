How it works

The service follows a configuration-driven approach: you define how each attribute should be treated (based on datatype and your data model). The anonymisation pipeline then applies the selected operators consistently.

3

The pipeline applies masking/generalisation/randomisation according to your config.

FAQ

Is the output truly “anonymous”?
Anonymisation is context-dependent. This tool applies transformations that can materially reduce privacy risk, but whether a dataset is anonymous depends on the broader context (available auxiliary data, threat model, and use case). If you need strict assurance, pair this with a formal risk assessment and governance process.
What formats are supported?
The focus is JSON and JSON-LD. JSON-LD support aims to preserve structure so that linked data workflows remain usable.
How do I provide good feedback?
The fastest way: open an issue with (1) a synthetic sample, (2) the steps you took, (3) expected output, and (4) the config you used (or a description of it). Create one here: new issue template.
Can I run this in my own environment?
Yes — the service is open source. Start from the repository: github.com/OwnYourData/anonymisation-service.
What data should I avoid uploading?
For testing this public instance, avoid secrets, health data, credentials, and any sensitive production payloads. Use synthetic samples whenever possible.
